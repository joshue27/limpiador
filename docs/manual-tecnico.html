🚀 Instalación

Opción 1: Script automático (Linux)

1

Clone el repositorio: git clone <url> limpiador && cd limpiador

2

Ejecute el instalador: chmod +x install.sh && ./install.sh

3

El script instala Docker (si no está), solicita credenciales y genera el archivo .env.

4

Construye las imágenes Docker y levanta los servicios.

5

Crea el usuario administrador automáticamente.

Tiempo estimado: 5-10 minutos (dependiendo de la velocidad de descarga de imágenes).

Opción 2: Instalación manual

1

Asegúrese de tener Docker y Docker Compose instalados.

2

Copie .env.production.example a .env y complete todas las variables.

3

Construya y levante: docker compose up -d --build

4

Espere a que la app esté lista: curl http://localhost:3000/api/health debe devolver {"status":"ok"}.

5

Cree el admin: curl -X POST http://localhost:3000/api/dev/bootstrap-admin -H "Content-Type: application/json" -d '{"email":"admin@dominio.com","password":"Contraseña123!"}'

Importante: El endpoint /api/dev/bootstrap-admin solo funciona cuando no hay usuarios en la base de datos. Después del primer admin, se desactiva automáticamente.

Instalación en Windows

1

Instale Docker Desktop desde docker.com.

2

Clone el repositorio o descargue el código fuente.

3

Ejecute PowerShell como administrador y ejecute: .\install.ps1

4

Siga las instrucciones interactivas.

Flujo de una conversación de WhatsApp

1. Cliente envía mensaje → Meta API → Webhook (POST /api/webhooks/whatsapp)
2. Next.js recibe → verifica firma HMAC → guarda en PostgreSQL
3. Si es texto nuevo → ejecuta routeInboundTextMessage()
4. Si es UNASSIGNED → envía menú automático (sendRoutingMenu)
5. Menú → cliente responde número → asigna a departamento (DEPARTMENT_QUEUE)
6. Operador ve conversación → reclama (CLAIMED) → responde vía WhatsApp API
7. Adjuntos → encola descarga → Worker descarga → guarda en media/

Flujo de una campaña

1. Admin crea borrador → agrega contactos (CSV o todos)
2. Admin lanza o programa → estado SENDING o QUEUED
3. Worker: enqueueCampaignSend() por cada destinatario → BullMQ
4. Worker procesa trabajo → sendTemplate() con variables resueltas
5. Éxito → SENT | Error → FAILED (3 reintentos)
6. Último destinatario → estado COMPLETED

Certificado SSL (recomendado)

WhatsApp Webhook requiere HTTPS. Use Certbot con Nginx:

# Instalar Certbot
sudo apt install certbot python3-certbot-nginx

# Obtener certificado
sudo certbot --nginx -d tudominio.com

# Renovación automática (ya configurada por Certbot)
sudo certbot renew --dry-run

Luego actualice APP_URL en .env a https://tudominio.com y configure el webhook en Meta con esa URL.

Health checks

Todos los servicios tienen health checks configurados:

  • Web: GET /api/health cada 15s (verifica DB + Redis)
  • PostgreSQL: pg_isready cada 10s
  • Redis: redis-cli ping cada 10s
  • Nginx: wget /api/health cada 15s

📦 Despliegue a producción

Checklist pre-lanzamiento

  1. ✅ .env configurado con valores de producción
  2. ✅ Token de WhatsApp permanente (no temporal)
  3. ✅ Dominio configurado con SSL (HTTPS)
  4. ✅ Webhook de WhatsApp verificado en Meta
  5. ✅ Plantillas aprobadas en Meta
  6. ✅ SMTP configurado (opcional pero recomendado)
  7. ✅ Backup de base de datos programado
  8. ✅ Logs rotativos configurados (ya incluido en compose)

Primer despliegue

# 1. Construir imágenes
docker compose build

# 2. Levantar servicios (escala según necesidad)
docker compose up -d --scale web=2

# 3. Verificar salud
curl http://localhost:80/api/health

# 4. Ver logs
docker compose logs -f

Actualización (nueva versión)

# 1. Detener servicios
docker compose down

# 2. Pull de cambios
git pull

# 3. Reconstruir y levantar
docker compose up -d --build --scale web=2

# 4. Aplicar migraciones (si las hay)
docker compose exec web npx prisma migrate deploy

Rollback

# Volver a un commit anterior
git checkout <commit-hash>
docker compose up -d --build --scale web=2

# O restaurar backup de BD
docker compose exec -T postgres pg_restore -U limpiador -d limpiador < backup.sql

💾 Backup y restauración

Backup de base de datos

# Backup completo
docker compose exec postgres pg_dump -U limpiador limpiador > backup.sql

# Backup comprimido
docker compose exec postgres pg_dump -U limpiador limpiador | gzip > backup.sql.gz

Restaurar base de datos

# Desde archivo SQL
docker compose exec -T postgres psql -U limpiador limpiador < backup.sql

# Desde archivo comprimido
gunzip -c backup.sql.gz | docker compose exec -T postgres psql -U limpiador limpiador

Backup de archivos multimedia

# Sincronizar a almacenamiento externo
rsync -avz media/ usuario@backup-server:/backups/limpiador/media/

# O usar Google Drive (configurar en la interfaz web)

Backup completo del sistema

# Todo en uno: BD + archivos
docker compose exec postgres pg_dump -U limpiador limpiador | gzip > backup_db.sql.gz
tar -czf backup_media.tar.gz media/ exports/

🔍 Solución de problemas

La aplicación no carga (error 502)

Causa: El servicio web no está listo o Nginx no puede conectarse.

Solución:

docker compose ps                    # Verificar qué servicios están caídos
docker compose logs web --tail=20     # Ver errores de la app
docker compose restart web nginx      # Reiniciar

Error "demasiados intentos" en login

Causa: Rate limiter bloqueando por múltiples intentos fallidos.

Solución:

docker compose exec redis redis-cli FLUSHDB   # Limpiar Redis
# O aumentar LOGIN_RATE_LIMIT_MAX en .env y reiniciar

Mensajes de WhatsApp no se envían

Causa: Token expirado o inválido.

Solución: Regenerar token en Meta Developers → actualizar .env → docker compose restart web worker.

Worker no procesa trabajos

docker compose logs worker --tail=20  # Ver si hay errores
docker compose exec redis redis-cli KEYS "bull:*"  # Ver colas
docker compose restart worker          # Reiniciar worker

Error de espacio en disco

df -h                                  # Ver uso de disco
du -sh media/ exports/                 # Ver tamaño de archivos
# Ajustar retención en Configuración → Retención de datos

Error "port already allocated"

docker compose down                   # Detener todo
sudo lsof -i :3000                     # Ver qué ocupa el puerto
sudo kill -9 <PID>                     # Liberar puerto
docker compose up -d                   # Reiniciar

🔒 Seguridad

Buenas prácticas implementadas

  • ✅ Contraseñas hasheadas con bcrypt (12 rounds)
  • ✅ Sesiones JWT con firma HMAC-SHA256
  • ✅ Rate limiting en login (por IP y email)
  • ✅ Rate limiting en descargas de archivos
  • ✅ CORS restringido a la URL configurada
  • ✅ Sanitización de paths para prevenir directory traversal
  • ✅ Content-Type validation en uploads
  • ✅ Auditoría de todas las acciones sensibles
  • ✅ Webhook de WhatsApp con verificación de firma HMAC
  • ✅ Secrets en variables de entorno (nunca en código)

Recomendaciones adicionales

  • 🔸 Usar firewall (ufw) para limitar puertos expuestos
  • 🔸 Configurar fail2ban para prevenir fuerza bruta
  • 🔸 Usar HTTPS con certificado válido (Let's Encrypt)
  • 🔸 Rotar SESSION_SECRET periódicamente
  • 🔸 Mantener dependencias actualizadas (npm audit)
  • 🔸 No exponer PostgreSQL ni Redis a Internet

📈 Escalado

Escalado horizontal (más instancias web)

# Escalar a 3 instancias
docker compose up -d --scale web=3

# Verificar balanceo
curl http://localhost/api/health  # Nginx round-robin

Escalado vertical (más recursos)

Edite docker-compose.yml para agregar límites:

services:
  web:
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G

Cuándo escalar

IndicadorAcción
CPU > 80% sostenidoAgregar instancias web
RAM > 80%Aumentar memoria o agregar instancias
Latencia > 500msVerificar queries lentas (índices)
Cola de campañas creceAumentar concurrency del worker
Disco > 80%Reducir retención o agregar almacenamiento
